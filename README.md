# lab 8: generative

## Задача
Генерация снежинок с помощью VAE 

## Предобработка данных
Данные взяты из датасета снимков multi-angle snowflake camera MASCDB. 
Описание датасета и библиотека для работы с ним: https://pymascdb.readthedocs.io/en/latest/index.html 

Датасет содержит снимки снежинок с трех разных камер и набор характеристик каждого экземпляра.

Для работы была оставлена одна камера и были отобраны снежинки, которые:
- являются планарными кристаллами (`snowflake_class_name: planar_crystal`) 
- не тают (`melting_class_name: dry`)
- не сильно заиневели (`riming_class_name: unrimed` или `rimed`)
- имеют максимальный диаметр `Dmax` (1500 самых больших)

Данные сохранены в одном файле `.npy` в виде массива формы `(N, 50, 50)`, где `N` – число изображений снежинок, а каждое изображение является одноканальным (grayscale).

#### SnowflakeDataset
Класс `SnowflakeDataset` реализует стандартный интерфейс PyTorch `Dataset`:
- при инициализации файл `.npy` загружается целиком в память;
- `__len__` возвращает количество изображений;
- `__getitem__` возвращает одно изображение, добавляя размерность канала, чтобы получить форму `(1, 50, 50)`.

Таким образом, каждый элемент датасета совместим с `Conv2d` слоями PyTorch

## Реализация VAE
Реализуется сверточный VAE с латентным пространством размерности `latent_dim = 16`
### Encoder
Энкодер представляет собой последовательность сверточных слоёв:
- `Conv2d(1 -> 32)` уменьшает разрешение с `50×50` до `25×25`;
- `Conv2d(32 -> 64)` уменьшает разрешение до `13×13`;    
- `Conv2d(64 -> 128)` уменьшает разрешение до `6×6`.

Между слоями используется `ReLU`.  
Энкодер преобразует изображение в компактное сверточное представление.

Размер выходного тензора энкодера вычисляется автоматически с помощью фиктивного входа `dummy`, чтобы избежать ручного подсчёта размерностей.

Далее сверточное представление разворачивается в вектор, из которого вычисляются:
- `mu` — среднее апостериорного распределения в латентном пространстве (распределение скрытой переменной при условии наблюдаемых данных). Отвечает за сдвиг распределения    
- `logvar` — логарифм дисперсии. Отвечает за масштаб шума

### Reparameterization

Для сэмплирования латентного вектора используется приём репараметризации:

$$z = \mu + \sigma \cdot \varepsilon,\quad \varepsilon \sim \mathcal{N}(0, I)$$

где

$$\sigma = \exp(0.5 \cdot \log \sigma^2)$$


### Decoder
Декодер зеркален энкодеру:
- латентный вектор `z` сначала проецируется полносвязным слоем обратно в сверточное представление;
- затем используется последовательность `ConvTranspose2d`, постепенно увеличивающая пространственное разрешение:
    - `6×6 -> 12×12 -> 24×24 -> 48×48`.
На выходе применяется `Sigmoid`, так как изображения нормированы в диапазон `[0, 1]`.

Выход декодера имеет размер `(1, 48, 48)`, поэтому для вычисления ошибки реконструкции входное изображение обрезается до центральной области такого же размера.

## Функция потерь

Используется стандартная функция потерь VAE:

$$\mathcal{L} = \mathcal{L}_{recon} + \beta \cdot \mathcal{L}_{KL}$$
### Reconstruction loss

Реконструкция считается через MSE:

$$\mathcal{L}_{recon} = \|x - \hat{x}\|_2^2$$​

Суммирование (`reduction='sum'`) используется для корректного масштаба ошибки относительно KL-дивергенции.

### KL-дивергенция

KL-дивергенция между апостериорным распределением и стандартным нормальным распределением:

$$\mathcal{L}_{KL} = -\frac{1}{2} \sum (1 + \log \sigma^2 - \mu^2 - \sigma^2)$$

KL-дивергенция делает латентные распределения похожими на стандартное нормальное.
Коэффициент `beta = 0.5` ослабляет регуляризацию и позволяет модели лучше фокусироваться на качестве реконструкций.


## Обучение модели

Обучение проводится в стандартном цикле PyTorch:
- модель переводится в режим `train`;
- для каждого батча:
    - выполняется прямой проход
    - считается функция потерь
    - выполняется backpropagation
    - обновляются параметры с помощью оптимизатора Adam (`learning rate = 1e-3`).

Средняя ошибка по эпохе логируется в TensorBoard с помощью `SummaryWriter`, что позволяет отслеживать динамику обучения и вовремя замечать переобучение или деградацию
